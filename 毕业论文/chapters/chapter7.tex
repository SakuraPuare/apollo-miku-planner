\chapter{时空可通行走廊与路径-速度协调}
\label{chap:st_corridor}

Apollo \texttt{PathBoundsDecider}以\texttt{IsStatic()}过滤将动态障碍物排除在路径决策之外，路径与速度两阶段之间也不存在显式的时空约束通道。本章以到达时间函数$\tau(s)$为基础构建时空可通行走廊，并将走廊约束以ST上界形式注入速度QP，与\texttt{SpeedBoundsDecider}原有输出完成双源融合。

\section{ST 路径-速度协调的源码缺陷}
\label{sec:problem_st}

\subsection{问题描述与源码定位}

Apollo的路径决策模块在确定障碍物的避让方向时，完全排除了动态障碍物。所有非静态障碍物，包括行驶中的车辆、行走的行人、骑行的自行车等，不参与\texttt{PathBoundsDecider}的边界构建，而是被完全交给速度规划阶段的ST图处理。这意味着路径规划阶段只感知静态障碍物，动态障碍物对路径决策过程不可见。

该问题的源码证据体现在\texttt{IsWithinPathDecider\-ScopeObstacle}函数中。代码\ref{lst:scope_obstacle}摘录了虚拟障碍物、IGNORE决策和非静态障碍物三类过滤条件，其中第三个分支是本文关注的直接证据。

\begin{lstlisting}[language=C++,caption={动态障碍物被路径决策排除},label={lst:scope_obstacle}]
// path_bounds_decider_util.cc: \texttt{IsWithinPathDeciderScopeObstacle}
bool \texttt{PathBoundsDeciderUtil}::\texttt{IsWithinPathDeciderScopeObstacle}(
    const Obstacle& obstacle) {
  // 虚拟障碍物不参与路径决策
  if (obstacle.IsVirtual()) {
    return false;
  }
  // 已有IGNORE决策的障碍物跳过
  if (obstacle.HasLongitudinalDecision() && obstacle.HasLateralDecision()
      && obstacle.IsIgnore()) {
    return false;
  }
  // 非静态障碍物或速度超过阈值的障碍物直接排除
  if (!obstacle.\texttt{IsStatic}() ||
      obstacle.speed() > FLAGS_static_obstacle_speed_threshold) {
    return false;  // 动态障碍物不参与路径决策
  }
  // TODO(jiacheng):
  // Some obstacles are not moving, but only because they are waiting
  // for red light (traffic rule) or because they are blocked by
  // others (social). These obstacles will almost certainly move in
  // the near future and we should not side-pass such obstacles.
  return true;
}
\end{lstlisting}

Apollo开发团队在该函数中留下了\texttt{TODO}注释，指出部分当前低速或静止的障碍物可能在短时间内重新运动，因而不应简单按静态障碍物处理。该注释说明Apollo源码中已经存在对静态/动态二分边界情形的工程疑虑，但它并不直接证明本文所讨论的所有动态障碍物路径决策缺失问题；本文的直接证据仍是前述\texttt{IsStatic()}过滤逻辑。

同样的过滤模式不止出现在\texttt{PathBoundsDecider}。路径决策器\texttt{PathDecider}的\texttt{MakeStaticObstacleDecision}在遍历障碍物列表的第一步同样调用\texttt{IsStatic()}过滤掉所有动态障碍物，再对剩余的静态障碍物执行STOP/NUDGE/IGNORE决策。Apollo路径规划阶段的两个核心决策模块都将动态障碍物完全排除在外，形成了系统性的路径-速度协调缺失。

\subsection{影响分析}

动态障碍物被排除在路径决策之外，产生以下影响。

第一，路径与速度规划之间缺乏协调。在Apollo的路径-速度解耦架构中，路径规划先于速度规划执行，速度规划在路径规划的结果上构建ST图。当动态障碍物不参与路径决策时，路径阶段可能生成一条直行路径，而速度阶段在该路径上发现动态障碍物横穿，不得不急刹或停车等待。如果路径阶段考虑了动态障碍物的预测轨迹，完全可以提前规划一条避让路径，在保持速度的同时安全通过。

第二，横穿行人在路径规划中完全不存在。考虑一个典型场景，一名行人以正向横向速度沿Frenet $+l$方向从车道中线向左侧移动。在路径规划阶段，由于该行人是动态障碍物，其\texttt{IsStatic()}返回false，\texttt{PathBoundsDecider}完全忽略其存在，路径规划生成的是一条沿车道中心线的直行路径。随后速度规划在ST图中检测到该行人的ST边界，只能通过减速或停车来避让。

具体地，设自车以$v_{ego} = \PedExampleVel$\,m/s行驶，前方$s = \PedExampleS$\,m处一名行人以$v_{lat} = \PedExampleLatVel$\,m/s沿$+l$方向横穿，当前位于车道中线$l = 0$处。自车到达该位置需$t = \PedExampleS/\PedExampleVel = \PedExampleTau$\,s，届时行人已横移$\PedExampleLatVel \times \PedExampleTau = \PedExamplePredL$\,m，移至$l = \PedExamplePredL$\,m处，超出道路左边界$l_{road}^+=\PedExampleRoadRight$\,m达\PedExampleOvershoot\,m，其右缘刚好脱离车道。如果路径规划阶段纳入行人的预测轨迹，可以基于行人在$t = \PedExampleTau$\,s时的预测位置$l = \PedExamplePredL$\,m计算路径边界，此时行人已不在车道中央，自车可从行人右侧安全通过，无需停车。但由于行人被\texttt{IsWithinPathDeciderScopeObstacle}过滤，路径阶段对此一无所知，速度阶段只能被迫停车等待。该场景的完整时变分析将在本章下一节中给出。

图\ref{fig:dynamic_exclusion}给出了该场景下行人时变位置与自车到达时刻的三帧对比。图\ref{fig:dynamic_exclusion:a}为$t=0$时刻，自车刚开始规划，行人距自车\PedExampleS\,m，当前位置在车道中线，但由于其被判定为动态障碍物，\texttt{PathBoundsDecider}的路径边界构造不会纳入该行人的预测轨迹。图\ref{fig:dynamic_exclusion:b}为$t=\PedExampleMidTau$\,s时刻，自车已行进\PedExampleMidS\,m，行人横移\PedExampleMidLatShift\,m进入车道左半侧，恰好落在自车直行路径上形成碰撞窗口。图\ref{fig:dynamic_exclusion:c}为$t=\PedExampleTau$\,s时刻，行人已越过车道左边界$\PedExampleOvershoot$\,m，自车到达原障碍位置时通行带打开。若Path阶段能够利用行人的预测轨迹，据其在自车到达该位置时已横移至车道左边界外这一事实，完全可以在规划初始即选择从行人右侧绕行或保持直行，无需在Speed阶段被动减速停车。

第三，动态障碍物的排除进一步加剧了问题一的影响。动态障碍物的排除使得\texttt{PathBoundsDecider}只能基于障碍物在$t = 0$时刻的静态快照做决策，无法利用时间维度的信息。例如，一辆缓慢移动的车辆在当前时刻$t = 0$占据车道中央，但$t = 2$s后将移至车道边缘甚至离开车道。如果纳入该动态障碍物的预测轨迹，\texttt{PathBoundsDecider}可以基于自车到达该$s$位置时障碍物的预测位置来决定避让方向，而非基于其当前位置。这种时间维度的利用可以显著减少问题一中BLOCKED判定的发生频率。

\subsection{路径决策信息未传递至速度规划}

动态障碍物被排除之外，ST层面还有一个同样严重的问题，即路径决策阶段产生的时空信息没有传递给速度规划阶段。

在Apollo的路径-速度解耦架构中，\texttt{PathBoundsDecider}输出的是横向路径边界$[l^-(s), l^+(s)]$，\texttt{SpeedBoundsDecider}基于该路径独立构建ST边界。两个模块之间的信息传递是单向的、有限的，速度规划知道路径在哪里，但不知道路径为什么在那里。

这种信息断层的后果在动态障碍物场景中尤为明显。考虑改进后的路径规划，路径决策可能基于\emph{行人将在$t = \PedExampleTau$\,s后移开}这一预判选择了一条通过方案，但\texttt{SpeedBoundsDecider}不知道这一时间约束的存在。如果速度规划恰好让自车在行人移开之前到达该位置，路径决策的前提条件就被违反了。

因此，ST层面的问题不仅是排除了动态障碍物，更本质的问题在于路径规划和速度规划之间缺乏时空约束的传递机制。改进方案需要让路径层面的时变决策以约束的形式传递给速度层面，确保两者的一致性。图\ref{fig:path_speed_breakage}展示Apollo解耦架构中Path阶段已掌握的时空决策依据如何被瓶颈截断，本章后续两节分别建立两条传递路径。

\section{到达时间估计与时变SL投影}
前节识别的ST层协调缺陷要求将SL层统一规划向带预测轨迹的时空情形推广。本节建立时变通行带所需的两项几何输入，以二阶运动学模型估计自车到达时间$\tau(s)$，并据此对动态障碍物施加时变SL投影，供后续走廊定义与速度约束融合使用。

\subsection{到达时间估计}
\label{sec:arrival_time_estimation}
时变通行带的关键在于将自车到达时间与障碍物的预测轨迹相耦合。令到达时间函数$\tau(s)$给出自车抵达纵向位置$s$的时刻。

最简单的估计是匀速假设$\tau_0(s) = (s - s_{ego}) / v_{ego}$。然而，匀速假设在自车正在加速或减速时会引入显著误差，足以导致通行带的时间有效性判断失误，定量比较参见图\ref{fig:arrival_time_compare}。

为提高精度，采用二阶运动学模型。根据匀加速运动方程$s - s_{ego} = v_{ego}\, t + \frac{1}{2} a_{ego}\, t^2$，解出到达时间

\begin{equation}
    \tau(s) = \frac{-v_{ego} + \sqrt{v_{ego}^2 + 2\, a_{ego}\, (s - s_{ego})}}{a_{ego}}
    \label{eq:arrival_time}
\end{equation}

其中$s_{ego}$、$v_{ego}$和$a_{ego}$分别为自车当前的纵向位置、速度和加速度，均可从Apollo的\texttt{VehicleState}模块直接获取。

该公式在两种边界情形下退化为更简单的形式。当$a_{ego} \to 0$时，对根号内进行Taylor展开$\sqrt{1 + x} \approx 1 + x/2$

\begin{equation}
    \tau(s) = \frac{v_{ego}\!\left(\sqrt{1 + \dfrac{2a_{ego}(s-s_{ego})}{v_{ego}^2}} - 1\right)}{a_{ego}} \;\approx\; \frac{v_{ego} \cdot \dfrac{a_{ego}(s-s_{ego})}{v_{ego}^2}}{a_{ego}} = \frac{s - s_{ego}}{v_{ego}}
    \label{eq:arrival_time_degenerate}
\end{equation}

即退化为匀速估计。当$v_{ego} \approx 0$且$a_{ego} \approx 0$即自车近似静止时，令$\tau(s) = +\infty$，算法退化为静态版本，这在物理上是合理的，静止的自车不会到达任何前方位置，因此障碍物的预测轨迹不产生时变效应。

在实际实现中，若上一规划周期已输出速度曲线$s_{prev}(t)$，且该曲线在规划时域内随$t$单调不减，可通过一维查表和线性插值求取首次满足$s_{prev}(t)\geq s$的时间，记为$\tau(s)$。该查表过程不要求$s_{prev}(t)$具有严格数学反函数；当曲线存在停车平台或目标$s$超出上一周期曲线覆盖范围时，分别取平台首次到达时间或回退至二阶运动学模型。二阶运动学模型作为首个规划周期即无历史速度曲线可用或速度曲线不可用时的回退方案。图\ref{fig:arrival_time_compare}以$v_{ego}=\ArrivalCompareEgoSpeed$\,m/s、$a_{ego}=\ArrivalCompareAccel$\,m/s$^2$的加速工况展示三种模型在前方\ArrivalCompareRange\,m范围内的估计曲线，以原始速度曲线反推为基准，在$s=\ArrivalCompareProbeS$\,m处匀速假设的误差约为\ArrivalCompareUniformError\,s，二阶运动学模型将残差压缩至约\ArrivalCompareSecondOrderError\,s。在加速度持续变化的工况下，二阶模型相比匀速假设精度提升约\ArrivalCompareImprovementPct\%，与历史速度曲线反推之间的残差则反映出常加速度假设无法完全捕捉非线性速度演化，因此实际工程中优先采用历史速度曲线反推，二阶模型仅作为首个规划周期或历史曲线不可用时的回退方案。

到达时间估计存在固有的循环依赖，$\tau(s)$用于计算时变通行带，通行带约束影响速度规划，而速度规划结果反过来决定了自车的实际到达时间。本文采用Apollo的周期性重规划机制，每\PlannerCycleMs\,ms执行一次，隐式处理这一耦合，每个周期使用上一周期的速度输出更新$\tau(s)$，在连续多个规划周期中实现近似收敛。这种单步延迟策略与Apollo路径-速度解耦框架的现有做法一致，其在城市道路工况下的实际效果已通过第\ref{chap:integration}章的实验验证。

式\eqref{eq:arrival_time}基于匀加速运动学假设，即自车在到达位置$s$的全程中保持加速度$a_{ego}$不变。然而，Apollo的\texttt{PiecewiseJerkSpeedOptimizer}输出的实际速度曲线$s(t)$远非常数加速度，在跟车、避让等典型工况下$s(t)$包含加速段、匀速段和减速段，加速度在各段之间由jerk惩罚项驱动而平滑过渡，呈现分段二次甚至更高阶的形态。式\eqref{eq:arrival_time}实质上是对这一非线性速度曲线在当前位置邻域的一阶Taylor近似。

本文的应对策略并非提高单次$\tau(s)$的估计精度，而是依赖Apollo每\PlannerCycleMs\,ms执行一次重规划的工程机制进行周期性修正。每个规划周期中，算法优先使用上一周期QP输出的速度曲线$s_{prev}(t)$进行单调查表并取首次到达时间，仅在无历史速度曲线、曲线非单调异常或查询点超出覆盖范围时回退至式\eqref{eq:arrival_time}的二阶运动学模型。在连续多个规划周期中，$\tau(s)$借助周期性修正逐步逼近真实到达时间，常加速度假设仅在首个周期或历史曲线不可用时生效。

对单个规划周期内常加速度假设引入的误差，可给出以下定性上界。$\tau(s)$的误差源于实际加速度$a(t)$与假设值$a_{ego}$之间的偏差。在加速度变化最剧烈的区间，即加减速过渡段，取典型城市工况参数$a_{\max} = \AccelMax$\,m/s$^2$与规划周期$\Delta t = \PlannerCycleMs$\,ms，单步速度预测误差约为$\Delta v \approx a_{\max} \cdot \Delta t = \VelocityPredError$\,m/s，对应的纵向位置预测误差约为$\Delta s \approx \frac{1}{2} a_{\max} \cdot \Delta t^2 \approx \PositionPredError$\,m。该误差量级远小于本文通行带构建所使用的安全裕度$\delta_i$，其取值范围为\DeltaRangeLow 至\DeltaRangeHigh\,m，因此常加速度近似在\PlannerCycleMs\,ms重规划周期下引入的位置误差对通行带边界判定不构成实质影响。

\subsection{动态障碍物的时变SL投影}
\label{sec:time_varying_sl_projection}
Apollo的Prediction模块为每个动态障碍物提供未来5\,s内的预测轨迹，以离散时间点序列$\{(x_i(t_j), y_i(t_j))\}$的形式给出。据此可算出障碍物在自车到达时刻$\tau(s)$的SL位置。

\begin{enumerate}
\item 在预测轨迹中查询时刻$\tau(s)$对应的障碍物位置，通过线性插值获取。

\item 将该位置从Cartesian坐标投影至参考线的Frenet坐标，得到$(s_i(\tau), l_i(\tau))$。

\item 结合障碍物的几何尺寸，计算时变SL边界$[l_i^{-}(\tau(s)), l_i^{+}(\tau(s))]$。

\item 若障碍物在$\tau(s)$时刻的纵向位置与$s$的距离超过其纵向尺寸的一半，即障碍物已移出该纵向截面的影响范围，则标记其在位置$s$处不活跃，不参与分组和间隙计算。
\end{enumerate}

这一机制使得即将移出自车路径的动态障碍物如横穿行人、变道车辆不再对路径边界构成约束，从而避免Baseline中因忽略动态信息而导致的不必要停车。

对于预测轨迹超出Prediction模块时间范围，通常为\PredictionHorizonSeconds\,s，的情形，采用保守策略，假设障碍物保持最后预测位置不变。这一假设的合理性在于，\PredictionHorizonSeconds\,s内自车以城市道路常见速度，取\CitySpeedMin 至\CitySpeedMax\,m/s，可行驶\PredictionDistanceMin--\PredictionDistanceMax\,m，已覆盖绝大多数需要路径决策的场景。对于更远距离的障碍物，路径决策可以在后续规划周期中随着距离缩短而更新。

此外，对于从Prediction模块接收到多条预测轨迹的情况，例如车辆可能直行或变道时存在多条候选轨迹，当前实现取概率最高的轨迹。对多条轨迹分别计算时变投影并取并集的保守处理可以进一步降低漏判风险，代价是每个周期内$\tau(s)$的查询次数按轨迹数线性增加。图\ref{fig:tvsl_pipeline}归纳了从Prediction模块原始轨迹到最终活跃障碍物集的四步数据流，到达时间$\tau(s)$在三个步骤中被共同引用；该步骤输出的活跃障碍物集即第\ref{chap:group}章定理\ref{thm:max_gap}所需的输入。

\section{时变间隙与走廊边界}
将SL投影沿时间维度延展，可在ST平面析出自车与各障碍物的时变可通行间隙，并合成统一的ST走廊。本节给出时变间隙的几何定义，再由其推导走廊上下边界与对应的速度约束。

\subsection{时变间隙与ST走廊定义}
\label{sec:time_varying_corridor_definition}
基于第\ref{sec:arrival_time_estimation}节的到达时间$\tau(s)$与第\ref{sec:time_varying_sl_projection}节的时变SL投影，本节将第\ref{chap:group}章静态间隙优化推广至动态情形，再把推广结果经$\tau(s)$抬升得到ST平面上的二维可行域。本节交付两项定义。其一是作为$s$函数的时变间隙$g_p(s)$与全路径最优分割$p^*$，逐$s$截面给出空间维度的通行性判定。其二是时空可通行走廊$\Omega$，将SL通行性沿$\tau(s)$抬升至ST平面，作为第\ref{sec:corridor_speed_constraint}节速度规划约束的几何载体。

对于连通分量内按$u_i(\tau(s))$排序的$k$个障碍物，先定义前缀左安全边界$V_p(s)=\max_{1\leq q\leq p}v_{(q)}(\tau(s))$，时变间隙定义为

\begin{equation}
    g_p(s) = \begin{cases}
        u_{(1)}(\tau(s)) - l_{road}^{-}(s) & p = 0 \\[0.3em]
        u_{(p+1)}(\tau(s)) - V_p(s) & p = 1, \ldots, k-1 \\[0.3em]
        l_{road}^{+}(s) - V_k(s) & p = k
    \end{cases}
    \label{eq:time_varying_gap}
\end{equation}

第\ref{chap:group}章定理\ref{thm:max_gap}在每个$s$截面上仍然成立。这是因为引理\ref{lem:continuous_partition}的连续分割证明和定理\ref{thm:max_gap}的直接代入验证不要求$u$和$v$是常数；只要在每个$s$处按$u_i(\tau(s))$排序，并用前缀最大值$V_p(s)$表达左侧约束边界，即可得到该截面的最大间隙。该结论是逐截面的精确结果；当要求沿一段$s$使用同一个组级分割时，仍需采用第\ref{sec:grouping_connected}节所述保守包络或枚举固定分割。

全路径的最优通行带宽度定义为

\begin{equation}
    W^{*} = \max_{p} \min_{s} g_p(s)
    \label{eq:time_varying_opt}
\end{equation}

该max-min问题要求在所有$s$截面上同时选择一个固定的组级分割，使得最窄处的间隙最大。在排序不随$s$变化且所有障碍物共同活跃的简化情形下，精确求解可枚举$k+1$个$p$值，对每个$p$遍历所有离散$s$求最小间隙，复杂度为$O(kK)$，其中$K$为离散点数。对一般链式连通分量或排序随时间变化的情形，单个$p$不再天然对应同一组障碍物集合。算法\ref{alg:optimal_band}在实际实现中采用第\ref{sec:grouping_connected}节的保守包络处理，以同一分量内所有障碍物同时施加约束作为充分可行条件，在包络上计算间隙并选取$p^{*}$。因此该近似保证安全可行，但不声称对一般时变链式分量给出全局最优解。

与静态版本的关键差异在于动态障碍物的$u_i(s)$和$v_i(s)$随$s$变化，因为$\tau(s)$变化导致查询的预测时刻不同，使得间隙在不同$s$处可能打开或关闭。例如，横穿行人在$s$较小处自车尚未到达时间隙为零，但在$s$较大处到达时行人已移走间隙打开。

至此空间维度上每个$s$截面的通行性已由$g_{p^*}(s) \geq W_{ego}$判定，把这一判定结果沿时间维度$\tau(s)$抬升，即在ST平面上构成本节定义的第二项对象，时空可通行走廊

\begin{equation}
    \Omega = \left\{(s, t) \;\middle|\; g_{p^*}(s) \geq W_{ego},\;\; t \geq \tau(s) \right\}
    \label{eq:st_corridor}
\end{equation}

其中第一个条件要求位置$s$处的最大间隙足以容纳自车宽度，第二个条件要求自车的到达时间不早于$\tau(s)$。$\Omega$的物理含义是，自车的ST轨迹$s(t)$只要完全落在区域$\Omega$内，就能保证在每个纵向位置都有足够的横向通行空间。

由式\eqref{eq:st_corridor}可见，$\Omega$在$s$维度由$g_{p^*}(s)\geq W_{ego}$判定通行与否，在$t$维度由$\tau(s)$作为下沿。若某位置$s_k$的最优间隙不足以容纳自车，整条纵向列$\{s_k\}\times \mathbb{R}_+$被从$\Omega$中排除，形成$s$方向的禁行切片。对于静态障碍物，$g_{p^*}(s)$仅取决于几何投影；对于动态障碍物，$g_{p^*}(s)$由自车抵达$s$的查询时刻$\tau(s)$决定，若自车到达$s_k$时障碍物恰好未移开，$s_k$列禁行，若已移开则打开。禁行切片在$\Omega$中呈现为$s$方向的水平缺口。

图\ref{fig:corridor_gen}以三阶段递进形式展示$\Omega$的构建过程，从单截面判定起步、沿$s$推广为剖面、最终升至ST平面。构造场景设道路宽$l\in[\CorridorRoadLeft,\CorridorRoadRight]$、$W_{ego}=\CorridorEgoWidth$\,m、$v_{ego}=\CorridorEgoSpeed$\,m/s，即$\tau(s)=s/\CorridorEgoSpeed$，含静态偏置障碍A位于$s\in[\CorridorStaticSMin,\CorridorStaticSMax]$、$l\in[\CorridorStaticLMin,\CorridorStaticLMax]$，及动态宽体障碍D，于$t=0$时占据$s\in[\CorridorDynamicSMin,\CorridorDynamicSMax]$、$l\in[\CorridorDynamicLMin,\CorridorDynamicLMax]$，以$v_{lat}=+\CorridorDynamicLatVel$\,m/s朝左横向移动。

\subsection{走廊边界与速度约束}
\label{sec:corridor_speed_constraint}
走廊$\Omega$的边界定义了自车ST轨迹的可行域。SL空间中的通行带$\mathcal{C} = \{(s, l) \mid l^{-}(s) \leq l \leq l^{+}(s)\}$是在到达时间函数$\tau(s)$的假设下计算的，其有效性依赖于一个关键前提，即自车的实际到达时间不早于$\tau(s)$

\begin{equation}
    t_{arrive}(s) \geq \tau(s), \quad \forall\, s
    \label{eq:validity_condition}
\end{equation}

若自车提前到达某位置$s_k$，例如因加速超过了$\tau(s)$的估计，则动态障碍物可能尚未移至预测位置，通行带在该处可能无效。该有效性条件可转化为速度规划的线性约束。对于每个依赖动态障碍物移开的关键位置$s_k$，要求

\begin{equation}
    s(t) \leq s_k, \quad \forall\, t < \tau(s_k)
    \label{eq:speed_constraint}
\end{equation}

式\eqref{eq:speed_constraint}的物理含义是，自车不得在$\tau(s_k)$时刻之前到达位置$s_k$，从而为动态障碍物的移开预留足够时间。从ST走廊的视角看，该约束正是走廊$\Omega$左边界在$s = s_k$处的数学表达，要求自车的ST轨迹不穿越走廊边界。更一般地，当存在多个关键位置$s_1, s_2, \ldots$时，各约束点$(\tau(s_k), s_k)$的集合构成了走廊$\Omega$左边界的离散采样。走廊约束的最终效果是要求QP的最优速度曲线$s^*(t)$始终位于走廊$\Omega$之内

\begin{equation}
    \left(s^*(t_j),\; t_j\right) \in \Omega, \quad j = 0, 1, \ldots, K
    \label{eq:trajectory_in_corridor}
\end{equation}

\section{走廊约束与速度规划融合}
本节将上一节得到的ST走廊边界与速度约束注入Apollo \texttt{PiecewiseJerkSpeedOptimizer}，与\texttt{SpeedBoundsDecider}原有输出在QP层完成双源融合，补齐路径-速度协调机制。

\subsection{走廊约束的速度规划融合}

\subsubsection{Apollo速度QP问题接口}
Apollo的\texttt{PiecewiseJerkSpeedOptimizer}在等时间间隔$\Delta t=0.1$\,s上以纵向位置$s_j$、速度$\dot{s}_j$、加速度$\ddot{s}_j$为决策变量，目标函数惩罚位置/速度跟踪误差与加加速度，约束包含运动学等式约束、物理限制以及由\texttt{SpeedBoundsDecider}给出的ST边界

\begin{equation}
    s_j^{lb,\text{SBD}} \leq s_j \leq s_j^{ub,\text{SBD}}, \quad j = 0, 1, \ldots, K
    \label{eq:st_bounds}
\end{equation}

QP问题最终化为

\begin{equation}
    \min_{\mathbf{x}} \;\frac{1}{2}\,\mathbf{x}^T \mathbf{H}\, \mathbf{x} + \mathbf{f}^T \mathbf{x} \quad \text{s.t.} \quad \mathbf{A}_{eq}\, \mathbf{x} = \mathbf{b}_{eq}, \quad \mathbf{x}^{lb} \leq \mathbf{x} \leq \mathbf{x}^{ub}
    \label{eq:qp_compact}
\end{equation}

由OSQP求解。本文走廊约束的嵌入策略是仅修改$\mathbf{x}^{ub}$中对应分量的值，保持$\mathbf{H},\,\mathbf{A}_{eq}$与稀疏模式完全不变，从而在不改动\texttt{PiecewiseJerkSpeedOptimizer}内部数据结构的前提下完成集成。

\subsubsection{时间有效性约束的嵌入}
本文算法输出的速度约束集合$\mathcal{T} = \{(s_k, \tau_k)\}$，其中$\tau_k = \tau(s_k)$，每个元素要求自车在时间$t<\tau_k$时满足$s(t)\leq s_k$。具体取法上，对每个被时变SL投影纳入路径决策的动态障碍物$i$，取其SL投影首次进入自车候选走廊的纵向位置$s_i^{0}$作为关键位置，即$\mathcal{T} = \{(s_i^{0},\,\tau(s_i^{0}))\}_{i\in\mathcal{D}}$，其中$\mathcal{D}$为参与路径决策的动态障碍物集合。对$v_s\approx 0$的横穿行人等典型情形，$s_i^{0}$退化为障碍物在参考线方向的当前纵向位置，如第\ref{sec:crossing_ped_numeric_example}节示例中$s_{ped}=\PedExampleS$\,m；对沿参考线方向有显著$v_s$的车辆级障碍物，$s_i^{0}$取自车候选走廊与障碍物预测SL轨迹首次相交截面的$s$值。该取法是走廊$\Omega$左边界沿$s$轴的最稀疏采样，其物理含义是要求自车在每个动态障碍物进入走廊处的实际到达时间不早于SL阶段所假设的$\tau(s_i^{0})$，从而保留式\eqref{eq:validity_condition}的有效性前提；自车若在$t<\tau_k$时已超过$s_k$，则SL阶段基于该障碍物已移开假设构造的通行带不再可信。$\mathcal{T}$元素数等于动态障碍物数$|\mathcal{D}|$，远小于QP时间步数$K$，因而该约束集对QP稀疏结构的影响可忽略。

需要指出，对纵向占据范围较大的障碍物，如公交车、大型车辆，单点约束可能不足以覆盖$[s_i^{-}, s_i^{+}]$范围内的所有位置。一种改进策略是将约束点扩展为$\mathcal{T}' = \{(s_i^{-}, \tau(s_i^{-})),\, (s_i^{+}, \tau(s_i^{+}))\}$，即在障碍物纵向范围的两端分别采样，以保证QP在$[s_i^{-}, s_i^{+}]$区间内的任意位置均满足到达时间约束。该扩展仅将约束数量从$|\mathcal{D}|$增至$2|\mathcal{D}|$，对QP稀疏结构的影响仍可忽略。本文当前测试场景中的动态障碍物均为行人，纵向尺寸不足1\,m，单点采样已足够；对车辆级障碍物建议采用两端采样策略。

该约束直接转化为对ST边界上界的收紧

\begin{equation}
    s_j^{ub} \leftarrow \min\!\left(s_j^{ub},\; s_k\right), \quad \forall\, j: t_j < \tau_k, \quad \forall\, (s_k, \tau_k) \in \mathcal{T}
    \label{eq:tighten_ub}
\end{equation}

在ST图中，式\eqref{eq:tighten_ub}的几何意义如图\ref{fig:st_constraint}所示，对应第\ref{sec:crossing_ped_numeric_example}节横穿行人示例的具体数值$(s_k=\PedExampleSk\,\text{m},\,\tau_k=\PedExampleTauK\,\text{s})$。对于每个约束点，在时间轴$t < \tau_k$的区间内添加一条水平上界线$s \leq s_k$，禁止自车在该时间范围内超越位置$s_k$；速度曲线在约束点附近贴沿抵达后，行人随即让出，QP在平滑性惩罚下自然恢复至$v_{ref}$。

\subsubsection{横穿行人场景数值示例}
\label{sec:crossing_ped_numeric_example}
沿用第\ref{sec:problem_st}节提出的横穿行人场景，以下按数值逐步演示从时变通行带到速度规划的全链路。

自车以$v_{ego} = \PedExampleVel$\,m/s匀速行驶，取$a_{ego} \approx 0$，前方$s_{ped} = \PedExampleS$\,m处一名行人以$v_{lat} = \PedExampleLatVel$\,m/s从右向左横穿马路。行人当前位于车道中央，取$l = \PedExampleBoundaryL$\,m，宽度\PedExamplePedWidth\,m。道路宽\PedExampleRoadWidth\,m，$l_{road}^{-} = \PedExampleRoadLeft$\,m，$l_{road}^{+} = \PedExampleRoadRight$\,m，示例安全裕度取$\delta = \PedExampleDelta$\,m。

到达时间计算。$a_{ego} \approx 0$，由式\eqref{eq:arrival_time_degenerate}退化为匀速估计

\begin{equation*}
    \tau(s_{ped}) = \frac{\PedExampleS - 0}{\PedExampleVel} = \PedExampleTau\,\text{s}
\end{equation*}

时变SL投影。在$\tau = \PedExampleTau$\,s时，行人横向位移$\Delta l = \PedExampleLatVel \times \PedExampleTau = \PedExamplePredL$\,m，位于$l = \PedExamplePredL$\,m。时变SL边界为$[l_{ped}^{-}, l_{ped}^{+}] = [\PedExampleLeftBoundary, \PedExampleRightBoundary]$\,m，其中$l_{ped}^{-}=\PedExampleLeftBoundary$\,m为行人在$+l$方向上靠近参考线的一侧，对应Apollo源码中的\texttt{l\_lower}；$l_{ped}^{+}=\PedExampleRightBoundary$\,m为远离参考线的一侧，对应\texttt{l\_upper}。符号按Frenet正$l$为左的约定取值\footnote{本章保留\texttt{LeftBoundary}/\texttt{RightBoundary}命名以与上下文宏体系一致，其语义分别对应Apollo SL多边形的\texttt{l\_lower}与\texttt{l\_upper}。}。

间隙计算。单个障碍物，此时$k = 1$，计算两个间隙

\begin{align}
    g_0 &= u_{ped} - l_{road}^{-} = (\PedExampleLeftBoundary - \PedExampleDelta) - (\PedExampleRoadLeft) = \PedExampleGapZero\,\text{m} \notag \\
    g_1 &= l_{road}^{+} - v_{ped} = \PedExampleRoadRight - (\PedExampleRightBoundary + \PedExampleDelta) = \PedExampleGapOne\,\text{m} \notag
\end{align}

最大间隙$g_0 = \PedExampleGapZero$\,m，大于自车宽度$W_{ego} = \PedExampleEgoWidth$\,m，自车可从行人右侧即道路右半侧安全通过。Baseline的静态快照中行人位于$l = \PedExampleBoundaryL$，间隙为$g_0 = (\PedExampleBoundaryL - \PedExamplePedHalfWidth - \PedExampleDelta) - (\PedExampleRoadLeft) = \PedExampleBaselineGap$\,m $< W_{ego}$，通行带不可行。

该通行带依赖行人在$\tau = \PedExampleTau$\,s时已移至$l = \PedExamplePredL$\,m的预测位置，因此算法输出速度约束

\begin{equation*}
    \mathcal{T} = \{(s_k, \tau_k)\} = \{(\PedExampleSk\,\text{m},\; \PedExampleTauK\,\text{s})\}
\end{equation*}

将上述约束嵌入QP。设$\Delta t = \PedExampleDeltaT$\,s，时间步$t_0 = 0, t_1 = \PedExampleDeltaT, \ldots, t_{\PedExampleLastRestrictedIndex} = \PedExampleLastRestrictedTime$\,s均满足$t_j < \tau_k = \PedExampleTauK$\,s。由式\eqref{eq:tighten_ub}，对这些时间步收紧上界

\begin{equation*}
    s_j^{ub} \leftarrow \min(s_j^{ub},\; \PedExampleSk\,\text{m}), \quad j = 0, 1, \ldots, \PedExampleLastRestrictedIndex
\end{equation*}

对于$j \leq \PedExampleLastRestrictedIndex$，即$t_j \leq \PedExampleLastRestrictedTime$\,s，原有ST上界$s_j^{ub}$通常较大，无其他近距离障碍物时接近规划终点，故新约束$s_j \leq \PedExampleSk$\,m生效。对于$j \geq \PedExampleFirstFreeIndex$，即$t_j \geq \PedExampleFirstFreeTime$\,s $> \tau_k$时，上界不受影响。

在新增上界约束下，QP优化器在平滑性惩罚的驱动下，输出的速度曲线在接近行人位置时轻微减速，减速幅度远小于Baseline的急停方案，行人移开后速度平滑恢复至$v_{ref}$。Baseline由于\texttt{IsStatic()}将行人排除在路径决策之外，速度阶段只能对行人横穿的全程时段输出YIELD停车决策，停车等待时长与行人横穿整条车道的时间相当，通行效率显著低于本文方法。

图\ref{fig:corridor}从更宏观的视角对比改进前后的约束生成链路。Apollo Baseline的速度侧按\texttt{SpeedBoundsDecider}$\to$\texttt{PathTimeHeuristicOptimizer}$\to$\texttt{SpeedDecider}$\to$\texttt{PiecewiseJerkSpeedOptimizer}的顺序执行。\texttt{SpeedBoundsDecider}首先为每个障碍物生成ST boundary多边形，\texttt{PathTimeHeuristicOptimizer}在ST图上以DP启发式搜索枚举YIELD/OVERTAKE组合得到全局粗解并作为暖启动参考，\texttt{SpeedDecider}再据此对各boundary写入YIELD/OVERTAKE标签，最终由QP求解分段线性的$s_j^{ub}$约束下的光滑速度曲线。本文方法在SL层按最大间隙原则全局选定分割$p^\star$，将$n$个障碍物的横向可通行信息压缩为式\eqref{eq:time_varying_gap}定义的$g_{p^\star}(s)$，并通过式\eqref{eq:st_corridor}与$\tau(s)$联合生成$\Omega$。$\Omega$本身由$\tau(s)$左下沿与水平禁行切片组合而成，几何上为非凸集合；本文方法的凸性来源不在$\Omega$而在其沿离散时间步的逐时间步线性投影$s_j^{ub}$，这与Baseline在\texttt{PathTimeHeuristicOptimizer}给出YIELD/OVERTAKE组合后所得的线性盒约束在QP结构上一致，因此可在不改动\texttt{PiecewiseJerkSpeedOptimizer}内部数据结构的前提下嵌入。SL层$p^\star$的前置选择避免了速度侧再次枚举方向组合，QP直接由OSQP求解全局最优$s^*(t)$。

\subsection{与SpeedBoundsDecider的双源约束融合}
Apollo速度QP接收两路ST边界，一路来自速度规划阶段，一路来自路径规划阶段。前者由\texttt{SpeedBoundsDecider}基于已确定路径的碰撞检测给出$s_j^{ub,\text{SBD}}$与$s_j^{lb,\text{SBD}}$，属Apollo原有约束源；后者即本章新增的ST走廊上界$s_j^{ub,\text{corridor}}$，来自路径决策阶段对动态障碍物时变特性的预判。两者在QP层取最小值融合

\begin{equation}
    s_j^{ub} = \min\!\left(s_j^{ub,\text{SBD}},\;\; s_j^{ub,\text{corridor}}\right), \quad j = 0, 1, \ldots, K
    \label{eq:dual_source}
\end{equation}

两个约束源的信息互补。\texttt{SpeedBoundsDecider}描述path上各障碍物的到达时刻，其ST边界基于给定路径的碰撞关系计算；ST走廊约束描述path决策对应的时序前提，其信息来源于路径阶段对动态障碍物预测轨迹的分析。该融合正好弥补了本章前节所识别的ST层问题，即路径决策信息未传递至速度规划，走廊约束即为此前缺失的路径-速度约束传递环节。

具体表现为三种互补情形。
\begin{enumerate}
\item 当场景中仅含静态障碍物时，动态障碍物不参与间隙计算或已标记为不活跃，$\mathcal{T} = \emptyset$，走廊约束不生效，\texttt{SpeedBoundsDecider}单独起作用。

\item 当场景中含动态障碍物时，走廊约束提供了\texttt{SpeedBoundsDecider}无法独立推导的时间窗信息。以横穿行人为例，\texttt{SpeedBoundsDecider}虽利用预测轨迹生成ST boundary，但\texttt{SpeedDecider}的YIELD决策将行人横穿的整个时段视为禁行区，而走廊约束给出精确的等待截止时间$\tau(s_k)$，将过度保守的全时段让行替换为精确的时间窗约束。

\item 在静态与动态障碍物混合的场景中，两个约束源各自对不同的时间步起主导作用。\texttt{SpeedBoundsDecider}负责静态障碍物的ST边界，走廊约束负责动态障碍物的时间窗，两者取最小值后QP自动得到全局一致的速度曲线。
\end{enumerate}

选择这种集成方式的原因在于，\texttt{SpeedBoundsDecider}之后的\texttt{PathTimeHeuristicOptimizer}已在ST图中对YIELD/OVERTAKE组合做了全局DP搜索，不存在\texttt{PathBoundsDecider}那样因贪心独立决策导致的系统性失效。\texttt{SpeedBoundsDecider}本身不需要做类似的全局最优化改造，ST走廊约束在QP层面补充时变信息即可，对原模块改动最小。

\section{时空走廊与路径-速度协调仿真}
\label{sec:exp_ch7}

本节以两个含动态行人的场景验证时变SL投影与ST走廊机制的实际效果。场景三为单行人横穿，是动态障碍物的最简形态，完整覆盖$\tau(s)$估计、时变SL投影、走廊节点$\mathcal{T}$生成、速度QP上界收紧的全链路；场景四为行人加静态停车的混合构型，检验时变投影在动静混合条件下将行人纳入路径决策的协调效果。每个场景均设压力构型与可通过构型两组；压力构型中Baseline因缺乏时变投影而不可通行，用以检验该机制的必要性；可通过构型中两种方法均能通过，用以比较连续指标。

\subsection{场景设置与时变投影}

自车以$v_{ego}=\PedCmpEgoSpeed$\,m/s沿车道中线行驶，前方$s=\PedCmpObstacleS$\,m处一名行人以$v_{lat}=\PedCmpPedVel$\,m/s从右向左横穿，当前位于$l=\PedCmpObstacleL$\,m。Baseline按\texttt{IsStatic()}过滤将行人完全排除在path决策之外，仅在\texttt{SpeedBoundsDecider}阶段对行人横穿全程时段输出时间窗约束；MIKU将行人纳入\texttt{PathBoundsDecider}，按$\tau(s_{ped}^{-})=\PedCmpSMinus/\PedCmpEgoSpeed=\PedCmpTau$\,s查询行人预测位置$l(\tau)=\PedCmpPredL$\,m，识别出$\tau$时刻行人已横移至车道左侧，自车右侧间隙打开，path直接绕行通过。

\subsection{压力子场景下Baseline减速不足}

压力构型中行人横穿时机更紧迫，Baseline将行人排除在path决策之外，仅依赖\texttt{SpeedBoundsDecider}的时间窗约束减速。由于时间窗约束的保守性，速度QP虽然执行了减速但未能在行人清除前完全停车，最终仅行驶至$s=\MOneBaselineSEnd$\,m，未到达目标终点$s_{target}=24$\,m。MIKU将行人纳入\texttt{PathBoundsDecider}，按$\tau(s)$查询行人预测位置后path直接绕行，速度QP无需额外减速约束，自车以$\bar{v}=\MOneMikuAvgV$\,m/s到达终点。

实验结果与本章理论分析一致。场景三压力构型中Baseline仅依赖时间窗约束减速，未能到达终点；场景四压力构型中Baseline因未将行人纳入path决策而在动静混合条件下失效。两个场景中MIKU均通过时变SL投影将行人信息前置到路径边界，path阶段直接绕行，无需速度层面的保守减速。可通过构型中两种方法均到达终点，MIKU的收益体现在更小的纵向冲击。四组对照从数值层面验证了式\eqref{eq:st_corridor}定义的时空走廊与式\eqref{eq:tighten_ub}速度QP上界收紧机制的实际效果，时变投影不仅在极端时机下是通行的必要条件，在宽松时机下也能改善轨迹质量。

\section{本章小结}

本章定位了Apollo \texttt{PathBoundsDecider}经\texttt{IsStatic()}过滤将动态障碍物排除在路径决策之外、路径-速度两阶段缺乏时空约束通道的设计缺陷。以二阶运动学模型导出的到达时间函数$\tau(s)$为桥梁，将动态障碍物的预测轨迹映射至自车到达时刻的SL投影，继而构造满足$t\geq\tau(s)$可达性条件的时空可通行走廊。走廊边界以ST上界形式与\texttt{SpeedBoundsDecider}的输出在速度QP层双源融合，形成path截断隐式传递与时间清单$\mathcal{T}$显式传递两条互补通道，在不改动速度规划模块结构的前提下补齐了此前缺失的路径-速度约束传递。场景三单行人横穿与场景四行人加停车的仿真表明，Baseline在压力构型下因缺乏时变投影而不可通行，MIKU借助$\tau(s)$将行人前置入路径边界后均顺利通过。
