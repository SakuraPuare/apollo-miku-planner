\chapter{多因子威胁度与差异化安全距离}
\label{chap:threat}

Apollo \texttt{PathBoundsDecider}对所有障碍物采用同一套安全距离，路边静止的花坛与正对车道的行人共用相同缓冲。该写法忽略了障碍物在威胁程度上的真实差异。本章先从源码与几何两方面剖析这一局限，再按多因子威胁度评估给出差异化安全裕度$\delta_i$的分配方法。

\section{一刀切缓冲的源码逻辑}
\label{sec:problem_margin}

\texttt{PathBoundsDecider}在构建路径边界时需要在障碍物边缘外预留安全距离，使自车与障碍物之间保持足够横向间距。该距离统一由\texttt{GetBuffer\-Between\-ADCCenter\-AndEdge}函数计算。代码\ref{lst:buffer_calc}摘录了总安全距离计算、静态性过滤以及nudge缓冲分支，后续分析围绕其中几个固定阈值展开。

\begin{lstlisting}[language=C++,caption={安全距离统一计算与nudge缓冲的静态/非静态分支},label={lst:buffer_calc}]
// path_bounds_decider_util.cc
// 相关 gflags 默认值（planning_gflags.cc）
//   obstacle_lat_buffer              = 0.4 m
//   static_obstacle_nudge_l_buffer   = 0.3 m
//   nonstatic_obstacle_nudge_l_buffer = 0.4 m
//   static_obstacle_speed_threshold  = 0.5 m/s

// 所有障碍物统一调用此函数计算安全距离
double PathBoundsDeciderUtil::
    GetBufferBetweenADCCenterAndEdge() {
  double adc_half_width =
      VehicleConfigHelper::GetConfig()
          .vehicle_param().width() / 2.0;
  return (adc_half_width + FLAGS_obstacle_lat_buffer);
}

// --- 静态/非静态的判定条件 ---
if (!obstacle.IsStatic() ||
    obstacle.speed() > FLAGS_static_obstacle_speed_threshold) {
  return false;  // 非静态障碍物不参与路径决策
}

// --- 边界松弛中的静态/非静态缓冲分支 ---
// 非静态障碍物使用 nonstatic buffer
double new_buffer = std::max(FLAGS_ego_front_slack_buffer,
                             FLAGS_nonstatic_obstacle_nudge_l_buffer);
// ...
// 静态障碍物使用 static buffer
if (path_bound_point->is_nudge_bound[LEFT_INDEX]) {
  old_buffer = std::max(FLAGS_obstacle_lat_buffer,
                        FLAGS_static_obstacle_nudge_l_buffer);
}
\end{lstlisting}

Apollo对安全距离的处理存在两个层面的粗粒度问题
\begin{enumerate}
\item \texttt{GetBufferBetweenADCCenterAndEdge}函数对所有障碍物返回相同的总安全裕度，即半车宽加0.4\,m额外缓冲，完全不区分障碍物的类型、速度、横向偏移等特征。无论是远离车道的路锥、车道边缘的花坛，还是正对车道的违停车，均使用相同的安全距离。

\item Apollo虽然在\texttt{nudge\_l\_buffer}参数中区分了静态0.3\,m和非静态0.4\,m两个额外缓冲等级，但仅以0.5\,m/s的速度阈值做二分，低于该阈值视为静态，高于则视为非静态。该二分判定在 \texttt{PathBoundsDecider} 与 ObstacleNudgeDecider 的缓冲计算中通过 \texttt{!IsStatic() || speed() > FLAGS\_static\_obstacle\_speed\_threshold} 条件式生效。这种二分法无法区分运动方向随时突变的行人与轨迹可预测的同向行驶车辆，也无法区分碰撞时间短的正面接近障碍物与碰撞风险低的横向远离障碍物。
\end{enumerate}

\section{统一缓冲对通行空间的几何压缩}

下面用一个具体场景说明统一安全距离如何引发不必要的通行失败。定义净间距为道路边界到障碍物边缘之间扣除额外缓冲$d_{buf}$后的剩余宽度，自车能从某一侧通过当且仅当该侧净间距不小于自车宽度。按第\ref{chap:framework}章式\eqref{eq:safety_margin}，换算到自车参考点约束时的总安全裕度为$\delta = W_{ego}/2 + d_{buf}$，下文\DeltaBaseline\,m和\NarrowExampleMikuDelta\,m均指额外缓冲而非$\delta$。

场景设定如下，标准车道宽\NarrowRoadWidth\,m，横向范围$[l_{road}^-,\;l_{road}^+] = [\NarrowExampleRoadLeft,\;\NarrowExampleRoadRight]$\,m，其中$l<0$为道路右侧、$l>0$为道路左侧。车道中偏右位置有一个路锥，横向占据$[\NarrowExampleConeLeft,\; \NarrowExampleConeRight]$\,m，宽度\NarrowExampleConeWidth\,m。自车宽度$W_{ego} = \NarrowExampleEgoWidth$\,m。

Apollo统一缓冲下，取$d_{buf} = \DeltaBaseline$\,m。从左侧通过时，净间距为
\[
\NarrowExampleRoadRight - (\NarrowExampleConeRight) - \DeltaBaseline = \NarrowExampleApolloGap\,\text{m} < W_{ego} = \NarrowExampleEgoWidth\,\text{m}
\]
从右侧通过时，净间距为
\[
\NarrowExampleConeLeft - (\NarrowExampleRoadLeft) - \DeltaBaseline = \NarrowExampleApolloRightGap\,\text{m} < \NarrowExampleEgoWidth\,\text{m}
\]
两侧均不足以容纳自车，路径被判定为BLOCKED，自车停车等待。

差异化缓冲下，该路锥处于静止状态且远离车道中心，威胁较低，取$d_{buf} = \NarrowExampleMikuDelta$\,m。从左侧通过时，净间距为
\[
\NarrowExampleRoadRight - (\NarrowExampleConeRight) - \NarrowExampleMikuDelta = \NarrowExampleMikuGap\,\text{m} = W_{ego}
\]
恰好等于自车宽度，自车从左侧安全通过。仅因缓冲参数由\DeltaBaseline\,m降到\NarrowExampleMikuDelta\,m，净间距就从\NarrowExampleApolloGap\,m提升至\NarrowExampleMikuGap\,m，通行结果由BLOCKED翻转为可行。图\ref{fig:safety_margin_compression}对比了两种缓冲参数下的通行带差异。按障碍物威胁程度差异化分配安全距离的具体方法见下一节。

\section{多因子威胁度评估模型}
前两节已经指出Baseline在安全裕度维度上的次优性，式\eqref{eq:safety_margin}中的统一$\delta$不区分障碍物威胁程度，使低威胁障碍物过度压缩通行带。本节建立量化模型，为每个障碍物计算综合威胁度$\Theta_i$，作为下一节差异化安全裕度$\delta_i$的输入。

对第$i$个障碍物，其综合威胁度$\Theta_i \in [0, 1]$由五个因子加权求和得到

\begin{equation}
\begin{split}
    \Theta_i = {} & w_1 \cdot f_{TTC}(i) + w_2 \cdot f_{overlap}(i) \\
                  & + w_3 \cdot f_{vel}(i) + w_4 \cdot f_{type}(i) + w_5 \cdot f_{inter}(i)
\end{split}
    \label{eq:threat_score}
\end{equation}

权重默认取值为$w_1 = \ThreatWeightTtc$，$w_2 = \ThreatWeightOverlap$，$w_3 = \ThreatWeightVel$，$w_4 = \ThreatWeightType$，$w_5 = \ThreatWeightInter$。权重确定过程分两步，先依据各因子对多障碍物场景风险的区分度做定性排序得到初始权重，再在对称路锥、横穿行人等典型场景上观察威胁度排序是否符合驾驶直觉，据此微调至最终取值。TTC因子与交互密度因子对场景风险区分度最高，赋予较大权重；障碍物类型在同一场景中变化较小，赋予较小权重。各因子取值范围均归一化到$[0, 1]$，定义如下。

权重取值遵循$w_1 > w_5 > w_2 > w_3 > w_4$的定性排序，即TTC因子权重最大、障碍物类型因子权重最小，与驾驶风险评估领域的基本共识一致。权重灵敏度实验表明，在$\pm20\%$扰动下各场景的威胁度排序与差异化安全裕度$\delta_i$均保持稳定，最大偏差不超过\SensWorstAbsDev{}\,m，详见第\ref{sec:sensitivity_exp}节。

\subsection{碰撞时间因子$f_{TTC}$}
碰撞时间（Time-to-Collision, TTC）是衡量碰撞紧迫性的经典指标

\begin{equation}
    TTC_i = \begin{cases}
        \dfrac{s_i - s_{ego}}{\dot{s}_{ego} - \dot{s}_i} & s_i > s_{ego} \text{ 且 } \dot{s}_{ego} > \dot{s}_i \\[0.6em]
        +\infty & \text{其他情形}
    \end{cases}
    \label{eq:ttc}
\end{equation}

其中$s_i$为障碍物纵向位置，$\dot{s}$为纵向速度。上述分段定义同时处理了三种退化情形，分别是障碍物在自车后方（$s_i\leq s_{ego}$）、自车不会追上障碍物（$\dot{s}_{ego}\leq\dot{s}_i$）以及相对速度为零导致分母退化（$\dot{s}_{ego}=\dot{s}_i$），均不构成纵向碰撞威胁。TTC因子通过分段线性函数映射至$[0, 1]$

\begin{equation}
    f_{TTC}(i) = \begin{cases}
        1.0 & TTC_i \leq T_{crit} \\[0.3em]
        \displaystyle\frac{T_{max} - TTC_i}{T_{max} - T_{crit}} & T_{crit} < TTC_i < T_{max} \\[0.5em]
        0 & TTC_i \geq T_{max}
    \end{cases}
    \label{eq:f_ttc}
\end{equation}

其中$T_{crit} = \ThreatTtcCrit$\,s为临界碰撞时间。该阈值依据Minderhoud与Bovy\cite{minderhoud2001extended}对TTC分布的理论分析，碰撞时间低于\ThreatTtcCrit\,s时驾驶员已难以通过转向规避碰撞，因此将此区间设为最大威胁等级。$T_{max} = \ThreatTtcMax$\,s为最大关注时间，与Apollo ST图的时间范围一致，覆盖城市道路典型制动-跟随工况的规划视野。$TTC_i \leq \ThreatTtcCrit$\,s意味着碰撞迫在眉睫，威胁最大；$TTC_i \geq \ThreatTtcMax$\,s的障碍物已超出当前规划周期的关注范围。

\subsection{横向重叠率因子$f_{overlap}$}
横向重叠率衡量障碍物与自车行驶走廊在横向上的重叠程度

\begin{equation}
    f_{overlap}(i) = \frac{\max\!\left(0,\; \min(l_i^{+}, l_{ego}^{+}) - \max(l_i^{-}, l_{ego}^{-})\right)}{W_{ego}}
    \label{eq:f_overlap}
\end{equation}

其中$l_{ego}^{+} = l_{ego} + W_{ego}/2$、$l_{ego}^{-} = l_{ego} - W_{ego}/2$为自车走廊的横向边界。$f_{overlap} = 0$表示完全不重叠即障碍物在自车投影之外，$f_{overlap} = 1$表示完全重叠即障碍物正对自车。该因子反映碰撞的几何可能性，横向偏移较大的障碍物即使TTC较小，实际碰撞风险也低。当障碍物为动态时，横向占据区间$[l_i^{-}, l_i^{+}]$取自时变SL投影在自车预计到达时刻$\tau(s)$的预测位置，即$f_{overlap}(i, \tau(s))$，详见第\ref{chap:st_corridor}章式\eqref{eq:arrival_time}；静态障碍物则直接取$t=0$的观测位置。

\subsection{相对速度因子$f_{vel}$}
相对速度因子反映障碍物与自车的接近速度

\begin{equation}
    f_{vel}(i) = \sigma\!\left(\frac{v_{rel,i}}{v_{max}}\right), \quad \sigma(x) = \frac{1}{1 + e^{-5x}}
    \label{eq:f_vel}
\end{equation}

其中$v_{rel,i} = \dot{s}_{ego} - \dot{s}_i$为相对纵向速度，正值表示自车接近障碍物，$v_{max}$为道路限速。Sigmoid函数将相对速度平滑映射至$(0, 1)$，快速接近的障碍物即$v_{rel} \gg 0$时趋近1，同向远离的障碍物即$v_{rel} < 0$时趋近0。选择Sigmoid而非线性映射，是因为相对速度为零附近的区分度更重要，即低速接近和低速远离的差异，而极高相对速度间的差异不需要精细区分。

\subsection{障碍物类型因子$f_{type}$}
取值映射如下，不同类型障碍物的运动不确定性与受伤严重度差异显著，按下式取离散值

\begin{equation}
    f_{type}(i) = \begin{cases}
        \ThreatTypePed  & \text{行人、自行车等弱势道路使用者} \\
        \ThreatTypeVehicle  & \text{机动车，运动相对可预测} \\
        \ThreatTypeUnknownMovable  & \text{未知可动目标，按保守权重处理} \\
        \ThreatTypeStatic  & \text{静态障碍物，仅需基本缓冲} \\
        \ThreatTypeCone & \text{交通锥等小尺寸标识物，威胁极低}
    \end{cases}
    \label{eq:f_type}
\end{equation}

取值依据如下，该映射综合考虑运动不确定性与事故后果。行人和自行车的运动方向和速度随时可能突变，如突然横穿或急停回转，轨迹可预测性远低于机动车，需要最大安全裕度；机动车受道路与交通规则约束，轨迹相对可预测；静态障碍物无运动不确定性，仅保留基本缓冲；交通锥等小尺寸标识物体积小且固定在路面，碰撞后果轻微，赋予最低权重\ThreatTypeCone。同时，一般认为弱势道路使用者（Vulnerable Road Users, VRU）在同等碰撞能量下受伤严重度和致死率远高于车内乘员，规划器应对其施加额外保护性裕度，因此赋予行人和自行车最高权重\ThreatTypePed。

在Apollo感知映射方面，Apollo感知模块在\texttt{PerceptionObstacle}消息中提供枚举字段\texttt{type}，取值覆盖\texttt{UNKNOWN}、\texttt{UNKNOWN\_MOVABLE}、\texttt{UNKNOWN\_UNMOVABLE}、\texttt{PEDESTRIAN}、\texttt{BICYCLE}和\texttt{VEHICLE}六类。规划层直接查表即可完成从感知标签到$f_{type}$的映射，\texttt{PEDESTRIAN}和\texttt{BICYCLE}归入\ThreatTypePed 分支，\texttt{VEHICLE}归入\ThreatTypeVehicle 分支，\texttt{UNKNOWN\_MOVABLE}与\texttt{UNKNOWN}归入\ThreatTypeUnknownMovable 分支，\texttt{UNKNOWN\_UNMOVABLE}归入\ThreatTypeStatic 分支，交通锥等已知小尺寸标识物通过几何尺寸判别归入\ThreatTypeCone 分支。对未知可动目标采用中等偏上的保守权重是因为分类置信度不足时宁可过估威胁也不可低估，避免将潜在行人误判为低威胁物体。

合成方式上，$f_{type}$作为加权求和中的一项进入式\eqref{eq:threat_score}，与碰撞时间、横向重叠率、相对速度三项连续因子加权叠加，合成综合威胁度$\Theta_i$，再经式\eqref{eq:dynamic_delta}映射为差异化安全裕度$\delta_i$，在裕度分配上对VRU给予更多保护。

\subsection{交互密度因子$f_{inter}$}
交互密度因子量化障碍物$i$与周围其他障碍物的聚集程度，是本文威胁度模型中与分组策略直接相关的因子

\begin{equation}
    f_{inter}(i) = \frac{1}{N-1} \sum_{j \neq i} \mathbb{1}\!\left[d_{ij} < d_{cluster}\right] \cdot \frac{d_{cluster} - d_{ij}}{d_{cluster}}
    \label{eq:f_inter}
\end{equation}

其中$N$为障碍物总数，$d_{ij} = \sqrt{(s_i - s_j)^2 + (l_i - l_j)^2}$为障碍物$i$和$j$在SL空间中的欧氏距离，$d_{cluster} = \ThreatClusterRadius$\,m为聚类半径，$\mathbb{1}[\cdot]$为指示函数。
当$N=1$即场景中仅有一个障碍物时，令$f_{inter}(i) = 0$，因为不存在可交互的邻居。$f_{inter}$的物理含义是，若障碍物$i$周围有大量其他障碍物聚集，则该因子值高，表明局部交互密度大，需要更谨慎的避让裕度。孤立障碍物的$f_{inter} \approx 0$，密集车群中的障碍物$f_{inter}$接近1。

交互密度因子能区分3个分散的障碍物与3个聚集的障碍物两类场景，它们对应不同的避让策略。前者各自独立，安全裕度可适当放宽；后者构成密集障碍物组，留给自车的横向空间本就有限，需要更精确的安全边界。$f_{inter}$较高的障碍物通常属于同一连通分量，将在下一节的纵向重叠分组中被统一协调。

图\ref{fig:threat_factors}以子图形式给出了各因子的函数形态。$f_\text{TTC}$在$T_\text{crit}$以下取饱和值1，在$T_\text{max}$以上回到0，中间为线性过渡；$f_\text{overlap}$随重叠率线性增长；$f_\text{vel}$采用Sigmoid函数，对零相对速度附近的变化最为敏感；$f_\text{type}$为离散取值，反映不同障碍物类型的固有风险差异；$f_\text{inter}$中单个邻居的贡献随距离线性衰减至聚类半径$d_{cluster}$处归零。各因子均归一化到$[0,1]$，权重之和为1，综合威胁度$\Theta_i$也落在$[0,1]$区间，可直接作为式\eqref{eq:dynamic_delta}中安全裕度线性插值的参数。

\section{差异化安全裕度的合成机制}
\label{sec:dynamic_delta_synthesis}
在综合威胁度$\Theta_i$的基础上，先对障碍物边缘外的额外缓冲做差异化插值，再与自车半宽合成式\eqref{eq:safety_margin}中的总安全裕度

\begin{align}
    d_{buf,i} &= d_{buf,min} + (d_{buf,max} - d_{buf,min}) \cdot \Theta_i \notag \\
    \delta_i &= \frac{W_{ego}}{2} + d_{buf,i}
    \label{eq:dynamic_delta}
\end{align}

其中$d_{buf,max}$和$d_{buf,min}$分别为额外缓冲的上下界，对应的总安全裕度上下界为$\delta_{max} = W_{ego}/2 + d_{buf,max}$与$\delta_{min} = W_{ego}/2 + d_{buf,min}$。$d_{buf,max}$与Apollo原始\texttt{point\_extension}取值一致，$d_{buf,min}$取较小值以释放低威胁障碍物占用的横向空间，两者的具体取值由系统集成阶段的参数标定确定。

式\eqref{eq:dynamic_delta}在两端保持与Apollo相同的行为。CRITICAL障碍物$\Theta_i \to 1$，总安全裕度与Apollo原始方案相同或更大，安全性没有下降；MINOR障碍物$\Theta_i \to 0$，额外缓冲较小，总裕度仍包含$W_{ego}/2$，原本被不必要占用的横向空间得以释放。

相应地，式\eqref{eq:def_u}--\eqref{eq:def_v}中的关键量更新为差异化版本

\begin{align}
    u_i &= l_i^{-} - \delta_i \label{eq:def_u_dynamic} \\
    v_i &= l_i^{+} + \delta_i \label{eq:def_v_dynamic}
\end{align}

当$\Theta_i$对所有障碍物均取相同值时，式\eqref{eq:def_u_dynamic}--\eqref{eq:def_v_dynamic}退化为式\eqref{eq:def_u}--\eqref{eq:def_v}。差异化$\delta_i$不影响第\ref{chap:group}章引理\ref{lem:continuous_partition}、定理\ref{thm:max_gap}及引理\ref{lem:boundary_trim}的越界预处理规则成立，详细证明见该章。

\section{差异化安全裕度仿真}
\label{sec:exp_ch4}

本节在水马与交通锥混合构成的窄路对照场景上验证差异化安全裕度对横向通行空间的释放效果。左侧为static类水马，右侧为cone类交通锥，两类障碍物的尺寸与类型差异使多因子模型输出不同的$\Theta_i$与$\delta_i$。仿真运行于Python复现的Apollo Planning流水线，覆盖\texttt{PathBoundsDecider}至\texttt{PiecewiseJerkSpeedOptimizer}的完整阶段。Baseline模式下所有障碍物共用统一额外缓冲$d_{buf}=\DeltaBaseline$\,m，总安全裕度$\delta_{\mathrm{base}}=W_{ego}/2+\DeltaBaseline$\,m；MIKU模式按式\eqref{eq:dynamic_delta}对每个障碍物独立计算$d_{buf,i}$与$\delta_i$。本节设置两组子场景，压力构型将障碍物内收至Baseline不可通行的临界位置，用于验证差异化缓冲的必要性；可通过构型将左侧水马轻量外移使两种方法都能通过，在同一口径下比较连续指标。

下面以$s=\NarrowThreatProbeS$\,m处最近的一对障碍物为例，对比多因子威胁度评估对两类障碍物的区分效果。两者共享相同的$f_{TTC}=\NarrowThreatTtcFactor$（$TTC=\NarrowThreatTtcSeconds$\,s）、$f_{overlap}=\NarrowThreatOverlap$（均在自车横向投影之外）、$f_{vel}=\NarrowThreatVel$（均为静态）和$f_{inter}\approx\NarrowThreatInter$（互为对侧邻居），差异集中在障碍物类型因子$f_{type}$。

对右侧交通锥，$f_{type}=\NarrowThreatType$，属五级分类最低档，反映其小尺寸、固定在路面、碰撞后果轻微的物理特征。综合威胁度$\Theta_{cone}\approx\NarrowThreatTheta$，由式\eqref{eq:dynamic_delta}得$d_{buf,cone}\approx\NarrowThreatDelta$\,m，总安全裕度$\delta_{cone}\approx\NarrowThreatTotalDelta$\,m。

两类障碍物的$\Theta$差异$\NarrowBarrierThreatTheta-\NarrowThreatTheta=0.01$虽小，但在通行带已处于临界宽度的几何条件下，$\delta$的任何缩减都直接转化为可用横向空间。更关键的是，两者的$\delta$均显著低于Baseline的$\delta_{\mathrm{base}}=\NarrowBaselineTotalDelta$\,m，这一整体缩减来自多因子模型对“静态、侧方、无横向重叠”三重低威胁特征的综合判定。若场景中存在行人（$f_{type}=\ThreatTypePed$）或横向重叠的车辆（$f_{overlap}\gg 0$），$\Theta_i$将远大于当前值，$\delta_i$接近甚至超过Baseline的统一裕度，从而对高威胁目标保持充分保护。

\section{本章小结}

本章针对Apollo \texttt{PathBoundsDecider}安全裕度一刀切的设计局限，给出多因子威胁度评估与差异化安全距离机制。源码层面定位了\texttt{GetBuffer\-Between\-ADCCenter\-AndEdge}函数对所有障碍物返回相同安全距离的根源；几何层面量化了统一缓冲对通行带宽度的不必要压缩。算法层面构建了融合TTC、横向重叠率、相对速度、障碍物类型与交互密度五项因子的威胁度模型，将归一化威胁度先映射为差异化额外缓冲$d_{buf,i}$，再合成为总安全裕度$\delta_i$，低威胁障碍物释放横向空间，高威胁障碍物保持甚至增大裕度。章内数值仿真以水马与交通锥混合窄路为对照场景，多因子模型对两类障碍物输出不同的$\Theta_i$与$\delta_i$，压力子场景下Baseline因零宽通行带不可通过，MIKU按差异化裕度完成通行；在可通过子场景两者均到达终点，MIKU保持更宽的通行带与更小的横向加速度峰值。
