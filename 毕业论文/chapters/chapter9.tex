\chapter{总结与展望}

\section{研究工作总结}

本文以百度Apollo 11.0开源自动驾驶平台为基础，针对其路径规划模块在多障碍物密集场景下的决策失效问题，从源码分析、问题识别、算法设计到仿真验证展开研究，提出多障碍物交互运动学统一规划算法（Multi-obstacle Interaction-density Kinematic-prediction Unified Planner，MIKU）。主要完成了以下工作。

\begin{enumerate}
\item \textbf{对Apollo Planning模块进行了源码级分析}\quad{}梳理了Planning模块的Scenario-Stage-Task三级管理架构、PublicRoadPlanner中基于EM Planner思想的路径-速度解耦两阶段优化流程、ReferenceLine平滑机制、PathDecider决策逻辑、PathBoundsDecider的路径边界构建流程以及SpeedBoundsDecider的ST边界构建机制。通过对关键源码的逐行分析，确认了PathBoundsDecider在路径边界构建阶段的逐障碍物独立决策模式、统一安全裕度分配、动态障碍物排除机制，以及SpeedBoundsDecider与路径阶段之间单向且有限的信息传递特征，为后续问题识别和算法改进提供了依据。

\item \textbf{从三个维度识别了多障碍物场景下路径规划失效的瓶颈}\quad{}通过源码分析从安全距离、决策粒度和时空协调三个维度识别了三个问题。问题一，所有障碍物使用统一的安全裕度$\delta$，不区分威胁程度，低威胁障碍物不必要地压缩了通行带宽度，加剧了路径阻塞的发生概率。问题二，PathBoundsDecider对每个障碍物独立执行贪心避让方向选择，当多个障碍物分布在参考线两侧时，独立决策产生的矛盾避让方向导致路径边界上下界交叉，即$l^+ < l^-$，触发BLOCKED状态，迫使车辆执行不必要的停车，是导致规划失败的直接原因。问题三，动态障碍物在路径决策阶段被完全排除，即源码中通过\texttt{IsStatic()}过滤，使得路径边界无法反映动态障碍物的实际影响，且路径-速度之间缺乏时空约束传递通道，导致速度阶段只能基于静态快照做出过度保守的YIELD停车决策。三个问题的共同根源在于路径边界构建缺乏全局视野、分级机制和时间信息。

\item \textbf{建立了多障碍物统一规划的数学模型}\quad{}将多障碍物避让方向选择建模为组合优化问题$(\mathbf{d}^*, \tau^*) = \arg\max \min_s W(s, \mathbf{d}, \tau)$。引理1利用反证法证明最优避让方向分配在障碍物横向排序中具有连续性，将搜索空间从$O(2^k)$降至$k+1$个候选；定理1直接代入验证每个候选分割的通行带宽度恰好等于对应的横向间隙，将精确求解复杂度降为$O(k \log k)$。

\item \textbf{设计并实现了基于预测轨迹的时变障碍物分组算法}\quad{}在SL层，提出多因子威胁度评估模型，结合TTC、横向重叠率、相对速度、障碍物类型和交互密度五个因子量化威胁程度，分配差异化安全裕度$\delta_i$；通过扫描线算法检测障碍物的纵向重叠关系，将关联障碍物聚为连通分量，在组内按横向位置排序并计算最大间隙，执行组级全局最优避让方向决策。在ST层，引入基于二阶运动学模型的到达时间函数$\tau(s)$和预测轨迹，将动态障碍物纳入路径边界决策；将时变通行带的时间有效性条件投影至ST空间构建时空可通行走廊，其边界约束与SpeedBoundsDecider的ST边界在QP层面融合，补上了此前缺失的路径-速度约束传递通道。算法总复杂度为$O(n \log n)$，满足实时性要求。

\item \textbf{通过仿真实验验证了改进算法的性能}\quad{}基于全链路Python仿真平台设计了4个测试场景，覆盖单一动态障碍物、动静混合、多动态障碍物以及密集静态障碍物四类工况，从通行成功率、平均速度、最大加速度幅值、最大jerk幅值、到达时间和QP求解耗时六个维度对Baseline与MIKU进行定量对比。实验结果表明，MIKU将平均巡航速度从\MSummaryBaselineAvgV\,m/s提升至\MSummaryMikuAvgV\,m/s，提升\MSummarySummaryVImprovement\%；通行成功率从\MSummaryBaselinePassCount{}个场景提升至\MSummaryMikuPassCount{}个场景；轨迹平顺性指标$|a|_{max}$与$|j|_{max}$在所有通行场景中均低于Baseline；QP求解总耗时由Baseline的\MSummaryBaselineQpTotal\,ms降至\MSummaryMikuQpTotal\,ms，降幅约89\%，原因在于路径阶段提前避开冲突使速度阶段无需重度回退，整体满足Apollo的100\,ms规划周期要求。消融实验表明C5走廊注入在简单场景为鲁棒性保险阀而非常态收益项，复杂场景下方为正贡献。
\end{enumerate}

\section{主要创新点}

本文的主要创新点如下。

\begin{enumerate}
\item \textbf{将多障碍物避让方向选择建模为最大通行带宽度优化问题，并证明了最大间隙策略的最优性}\quad{}引理1通过反证法证明最优避让方向分配在障碍物横向排序中具有连续性，将指数级搜索空间降至线性枚举；定理1证明每个候选分割的通行带宽度恰好等于对应的横向间隙，从而将最优避让策略等价为选择最宽间隙。该结论给出了$O(k \log k)$的精确求解算法，同时从数学上说明了Apollo原始贪心策略在多障碍物场景下失效的原因。

\item \textbf{提出了基于扫描线的时变障碍物分组算法}\quad{}通过纵向重叠检测将关联障碍物聚为连通分量，在组级别应用全局最优决策，消除了逐障碍物贪心策略产生矛盾避让方向的问题。不同连通分量的决策互不干扰，在保证全局最优性的同时保持了算法的局部性和可扩展性。

\item \textbf{设计了面向通行带优化的威胁度分级与动态安全距离机制}\quad{}结合TTC、横向重叠率、相对速度、障碍物类型和交互密度五个因子量化威胁程度，据此分配差异化安全裕度$\delta_i$。该机制的关键在于将差异化$\delta_i$与最大间隙算法结合。引理1和定理1对$\delta$是否为常数不做假设，差异化$\delta_i$可直接代入最优求解框架。低威胁障碍物因此释放通行空间，在不降低安全性的前提下扩大通行带宽度。

\item \textbf{引入到达时间函数和预测轨迹，将动态障碍物纳入路径边界决策，并建立ST走廊与速度规划的协调机制}\quad{}Apollo PathBoundsDecider在路径边界构建阶段完全排除动态障碍物，本文利用预测轨迹计算障碍物在自车到达时刻的预期位置，使路径边界能反映动态障碍物的时变特性。对于正在横穿的行人等场景，时变投影能捕捉障碍物移开后间隙打开的可能性，避免过度保守的停车决策。进一步将时变通行带投影至ST空间构成时空可通行走廊，走廊边界转化为速度QP的上界约束，与SpeedBoundsDecider的ST边界融合，在不修改速度规划模块结构的前提下实现路径-速度协调，解决了第六章指出的路径-速度信息断层问题。
\end{enumerate}

\section{研究局限性}

本文工作存在以下局限。

\begin{enumerate}
\item \textbf{仿真平台的等效性}\quad{}本文的数值仿真基于Python全链路复现Apollo Planning流水线，虽然严格对齐了PathBoundsDecider、SpeedBoundsDecider等七个核心阶段的算法逻辑，QP子问题均采用OSQP求解，但Python实现与Apollo C++原生系统在数值精度、内存管理和执行时序上仍存在差异。仿真平台的等效性已通过指标对照进行了初步验证，但尚未在Apollo C++代码库中进行端到端的集成测试，两者之间的一致性有待进一步确认。

\item \textbf{测试场景的覆盖度}\quad{}本文设计了4个代表性测试场景，分别覆盖单一动态障碍物、动静混合、密集静态障碍物和多障碍物连通分量四类典型工况。这些场景为手工构造的确定性场景，尚未涵盖交叉路口多方向冲突、高速公路汇流、连续变道等更复杂的实际交通情形，场景覆盖度有限。后续需要引入随机化场景生成与大规模蒙特卡洛测试，以更全面地评估算法的鲁棒性。

\item \textbf{威胁度权重的标定方法}\quad{}多因子威胁度模型中TTC、横向重叠率、相对速度、障碍物类型与交互密度五个因子的权重基于定性分析与工程经验确定，缺乏系统化的标定方法。当前权重在4个测试场景中表现良好，但在更广泛的场景分布下是否仍然合理尚不确定。后续可通过敏感性分析或数据驱动的参数优化方法对权重进行系统标定。

\item \textbf{单模态预测轨迹的假设}\quad{}本文的时变SL投影取Prediction模块输出的概率最高的单条预测轨迹作为障碍物未来位置的依据。对于行为意图明确的场景此假设合理，但在多模态预测场景下，例如前车可能直行也可能变道，单条轨迹无法反映行为的不确定性，可能导致路径决策对低概率但高风险的备选轨迹缺乏防范。
\end{enumerate}

\section{未来研究方向}

后续研究可沿以下方向继续推进。

\begin{enumerate}
\item \textbf{交互式预测与博弈感知决策}\quad{}本文的时变障碍物投影建立在Prediction模块单次前向输出的预测轨迹之上，能刻画行人横穿、车辆并线等动态过程，但尚未显式建模自车决策对周车行为的反作用。密集交互场景中，障碍物的实际轨迹会随自车的避让方向与加减速节奏改变，开环预测因此存在固有偏差。后续可引入博弈论方法，例如迭代最优响应，将自车与周车建模为双向耦合系统，把自车候选行为反馈进预测过程形成闭环交互推断；进一步将差异化威胁度$\delta_i$从静态评估扩展为交互均衡下的动态量，使最优分配同时反映几何约束与交互代价。

\item \textbf{到达时间与速度规划的迭代优化}\quad{}本文采用的二阶运动学到达时间模型相较匀速假设已有明显改进，多数工况下能保证时空对齐精度，但仍属于单次前向估计；当QP输出的速度曲线与$\tau(s)$估计偏离较大时，例如需要大幅减速的场景，时变投影可能产生偏差。后续可将到达时间估计改为路径与速度的迭代求解，首次以匀速假设计算通行带，基于该通行带生成路径和速度曲线后，用更新的速度剖面重新估计到达时间，重复迭代至收敛。需要重点解决的问题是迭代收敛性的理论保证与计算预算控制，使算法在满足实时性的同时获得更精确的时空匹配。

\item \textbf{纵向部分重叠的精确处理}\quad{}本文对$s$方向仅部分重叠的障碍物组采用公共截面间隙作为通行带宽度的保守估计，障碍物纵向间距较小时该处理合理，但某些特殊几何构型下未能完全释放可用空间。后续可将组级决策扩展为沿$s$方向逐截面求解，离散化生成多组间隙序列后在序列上求全局最优分配，更精确地描述通行带沿纵向的变化。该方向的主要难点是精度提升与计算开销的折衷，需要研究自适应离散粒度策略与分支限界加速技术。

\item \textbf{学习辅助的间隙选择}\quad{}当多个候选间隙宽度接近时，最大间隙策略虽在几何意义上最优，但未显式建模变道代价、上下游道路拓扑、驾驶习惯等因素。后续可引入学习方法，例如基于历史驾驶数据训练的间隙评估网络，对候选间隙综合评分，在宽度近似的情况下做出更符合人类驾驶偏好的选择。引理1与定理1对$\delta_i$的具体形式不做假设，因此学习模块可直接接入最大间隙框架。

\item \textbf{实车平台验证与工程化部署}\quad{}本文实验基于Apollo PNC仿真平台完成，覆盖了单一动态、动静混合、多动态以及密集静态四类工况。下一步计划将改进算法部署至搭载Apollo系统的实车平台，在真实交通环境中评估传感器噪声、感知延迟与执行误差对算法的影响，并完善极端工况下的降级策略与失效安全机制。需要重点关注感知误差在路径边界构建中的传播规律，以及规划-控制延迟下时变通行带的鲁棒性。
\end{enumerate}
