\chapter{总结与展望}

\section{研究工作总结}

本文以百度Apollo 11.0开源自动驾驶平台为研究对象，针对其路径规划模块在多障碍物密集场景下的决策失效问题，从源码分析、问题识别、算法设计到仿真验证进行了系统性研究，提出群组感知时变通行带规划算法（Group-aware Time-varying Optimal Corridor，GTOC）。主要完成了以下工作。

\begin{enumerate}
\item \textbf{对Apollo Planning模块进行了源码级深度分析}\quad{}系统梳理了Planning模块的Scenario-Stage-Task三级管理架构、PublicRoadPlanner中基于EM Planner思想的路径-速度解耦两阶段优化流程、ReferenceLine平滑机制、PathDecider决策逻辑、PathBoundsDecider的路径边界构建流程以及SpeedBoundsDecider的ST边界构建机制。通过对关键源码的逐行分析，揭示了PathBoundsDecider在路径边界构建阶段的逐障碍物独立决策模式、统一安全裕度分配、动态障碍物排除机制，以及SpeedBoundsDecider与路径阶段之间单向且有限的信息传递特征，为后续问题识别和算法改进提供了可靠依据。

\item \textbf{从三个维度系统识别了多障碍物场景下路径规划失效的根本瓶颈}\quad{}通过源码分析从安全距离、决策粒度和时空协调三个维度识别了三个核心问题。问题一，所有障碍物使用统一的安全裕度$\delta$，不区分威胁程度，低威胁障碍物不必要地压缩了通行带宽度，加剧了路径阻塞的发生概率。问题二，PathBoundsDecider对每个障碍物独立执行贪心避让方向选择，当多个障碍物分布在参考线两侧时，独立决策产生的矛盾避让方向导致路径边界上下界交叉，即$l^+ < l^-$，触发BLOCKED状态，迫使车辆执行不必要的停车，是导致规划失败的直接瓶颈。问题三，动态障碍物在路径决策阶段被完全排除，即源码中通过\texttt{IsStatic()}过滤，使得路径边界无法反映动态障碍物的实际影响，且路径-速度之间缺乏时空约束传递通道，导致速度阶段只能基于静态快照做出过度保守的YIELD停车决策。三个问题的共同根源在于路径边界构建缺乏全局视野、分级机制和时间信息。

\item \textbf{建立了时变最优通行带的数学模型}\quad{}将多障碍物避让方向选择建模为组合优化问题$(\mathbf{d}^*, \tau^*) = \arg\max \min_s W(s, \mathbf{d}, \tau)$。通过引理1（最优分割的连续性），利用反证法证明了最优避让方向分配在障碍物横向排序中具有连续性，将搜索空间从$O(2^k)$降至$k+1$个候选；通过定理1（最大间隙最优性），直接代入验证了每个候选分割的通行带宽度恰好等于对应的横向间隙，将最优通行带的精确求解复杂度降为$O(k \log k)$。该数学框架为工程实现提供了严格的理论保证。

\item \textbf{设计并实现了基于预测轨迹的时变障碍物分组算法}\quad{}在SL层，提出多因子威胁度评估模型，结合TTC、横向重叠率、相对速度、障碍物类型和交互密度五个因子量化威胁程度，实现差异化安全裕度$\delta_i$的分配；通过扫描线算法检测障碍物的纵向重叠关系，将相互关联的障碍物聚为连通分量，在组内按横向位置排序并计算最大间隙，执行组级全局最优避让方向决策。在ST层，引入基于二阶运动学模型的到达时间函数$\tau(s)$和预测轨迹，将动态障碍物纳入路径边界决策；将时变通行带的时间有效性条件投影至ST空间，构建时空可通行走廊，其边界约束与SpeedBoundsDecider的ST边界在QP层面融合，建立了此前缺失的路径-速度约束传递通道。算法总复杂度为$O(n \log n)$，满足实时性要求。

\item \textbf{通过仿真实验全面验证了改进算法的性能}\quad{}基于全链路Python仿真平台设计了4个测试场景，覆盖单一动态障碍物、动静混合、多动态障碍物以及密集静态障碍物四类典型工况，从通行成功率、平均速度、最大加速度幅值、最大jerk幅值、到达时间和QP求解耗时六个维度进行Baseline与GTOC的定量对比。实验结果表明，GTOC将平均巡航速度从\MSummaryBaselineAvgV\,m/s提升至\MSummaryGtocAvgV\,m/s，提升\MSummarySummaryVImprovement\%；通行成功率从\MSummaryBaselinePassCount{}个场景提升至\MSummaryGtocPassCount{}个场景；轨迹平顺性指标$|a|_{max}$与$|j|_{max}$在所有通行场景中均低于Baseline；QP求解总耗时由\MSummaryBaselineQpTotal\,ms降至\MSummaryGtocQpTotal\,ms，原因在于路径阶段提前避开冲突使速度阶段无需重度回退，整体满足Apollo的100\,ms规划周期要求。
\end{enumerate}

\section{主要创新点}

本文的主要创新点归纳如下。

\begin{enumerate}
\item \textbf{将多障碍物避让方向选择建模为最大通行带宽度优化问题，并给出了最大间隙策略最优性的理论分析}\quad{}引理1通过反证法证明了最优避让方向分配在障碍物横向排序中具有连续性，将指数级搜索空间降维为线性枚举；定理1进一步证明了每个候选分割的通行带宽度恰好等于对应的横向间隙，从而将最优避让策略等价为选择最宽间隙这一直观操作。该理论结果不仅提供了$O(k \log k)$的精确求解算法，也从数学上解释了为何Apollo原始贪心策略在多障碍物场景下必然失效。

\item \textbf{提出了基于扫描线的时变障碍物分组算法}\quad{}通过纵向重叠检测将互相关联的障碍物聚为连通分量，在组级别应用全局最优决策，从根本上消除了逐障碍物贪心策略产生矛盾避让方向的问题。分组机制使得不同连通分量的决策互不干扰，在保证全局最优性的同时保持了算法的局部性和可扩展性。

\item \textbf{设计了面向通行带优化的威胁度分级与动态安全距离机制}\quad{}基于多因子评估方法，结合TTC、横向重叠率、相对速度、障碍物类型和交互密度五个因子量化威胁程度，据此分配差异化的安全裕度$\delta_i$。该机制的核心贡献不在于威胁度模型本身，而在于将差异化$\delta_i$与最大间隙算法相结合，引理1和定理1对$\delta$是否为常数不做假设，因此差异化$\delta_i$可直接代入最优求解框架。该设计使低威胁障碍物释放通行空间，在不降低安全性的前提下扩大通行带宽度。

\item \textbf{引入到达时间函数和预测轨迹，将动态障碍物纳入路径边界决策，并建立ST走廊与速度规划的协调机制}\quad{}弥补了Apollo PathBoundsDecider在路径边界构建阶段完全排除动态障碍物的不足，利用预测轨迹计算障碍物在自车到达时刻的预期位置，使路径边界能够反映动态障碍物的时变特性。对于正在横穿的行人等场景，时变投影能够捕捉到障碍物移开后间隙打开的可能性，避免过度保守的停车决策。进一步将时变通行带投影至ST空间，揭示其本质上构成时空可通行走廊，走廊边界转化为速度QP的上界约束，与SpeedBoundsDecider的ST边界形成双源融合，在不修改速度规划模块结构的前提下实现了路径-速度协调，解决了第七章所揭示的路径-速度信息断层问题。
\end{enumerate}

\section{研究局限性}

本研究仍存在以下不足。

\begin{enumerate}
\item \textbf{实验验证以仿真为主}\quad{}受限于实验条件，算法验证完全基于Apollo的PNC仿真环境，尚未在实车平台上进行测试。仿真环境无法完全模拟真实道路中的传感器噪声、感知延迟和执行误差等因素，算法在实际部署中的鲁棒性有待进一步检验。

\item \textbf{纵向部分重叠采用保守近似}\quad{}当障碍物在$s$方向仅部分重叠时，本文将整个连通分量作为一个整体处理，使用公共$s$截面上的间隙作为通行带宽度的估计。这种保守近似在障碍物纵向间距较小时是合理的，但在某些特殊几何构型下可能低估实际可用空间。

\item \textbf{到达时间估计未与速度规划进行迭代优化}\quad{}到达时间函数$\tau(s)$采用基于当前速度和加速度的二阶运动学模型，相比匀速假设已有显著改进，但仍属于单次前向估计，未与速度规划的输出进行迭代修正。当QP输出的速度曲线与$\tau(s)$的估计偏差较大时，例如需要大幅减速的场景，时变投影可能偏离实际情况。

\item \textbf{依赖预测轨迹的准确性}\quad{}动态障碍物的时变SL投影完全依赖Prediction模块输出的预测轨迹。在交互场景中，障碍物的行为可能因自车决策而改变，当前的开环预测无法捕捉这种耦合效应。

\item \textbf{算法的跨版本兼容性未经验证}\quad{}本文算法仅在Apollo 11.0版本上实现和验证，Apollo平台版本迭代较快，算法的跨版本兼容性有待进一步测试。
\end{enumerate}

\section{未来研究方向}

基于上述局限性，未来的研究可从以下方向展开。

\begin{enumerate}
\item \textbf{纵向部分重叠的精确处理}\quad{}对$s$方向部分重叠的障碍物组，可沿$s$方向离散化为若干截面，在每个截面独立求解最优间隙后取全局最小值。这相当于将一维的组级决策扩展为逐截面决策，能够更精确地刻画通行带的纵向变化特征，但需要在精度提升与计算开销之间取得平衡。

\item \textbf{到达时间与速度规划的迭代优化}\quad{}将到达时间估计从单次前向计算改为路径-速度的迭代求解，首次以匀速假设计算通行带，基于该通行带生成路径和速度曲线后，用更新的速度剖面重新估计到达时间，重复迭代至收敛。这种方法能够提高时变投影的准确性，关键在于保证迭代的收敛性和控制额外计算开销。

\item \textbf{学习辅助的间隙选择}\quad{}当多个间隙的宽度接近时，最大间隙策略给出的解虽然在通行带宽度上最优，但未考虑变道代价、上下游道路拓扑等因素。可引入学习方法，如基于历史驾驶数据训练的评估网络，对候选间隙进行综合评分，在宽度近似的间隙之间做出更符合驾驶习惯的选择。

\item \textbf{实车验证与工程化}\quad{}将改进算法部署到搭载Apollo系统的实车平台上，在真实交通环境中验证算法的有效性和实时性。重点关注感知误差传播、规划-控制延迟、极端工况下的降级策略等工程化问题，推动算法从仿真验证走向实际应用。
\end{enumerate}
