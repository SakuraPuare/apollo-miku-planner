## 一、课题目标与内容

### 1.1 课题总体目标

本课题以百度Apollo 11.0开源自动驾驶平台为研究基础，针对其Planning模块在多障碍物密集场景下路径规划失效的问题，进行源码级分析、数学建模、算法设计与仿真验证，最终提出一套完整的改进方案。

### 1.2 核心功能

本课题需要实现的核心功能包括：

**（1）Apollo Planning模块源码级深度分析功能**

对Planning模块的Scenario-Stage-Task三级管理架构、PublicRoadPlanner路径-速度解耦两阶段优化流程、PathBoundsDecider路径边界构建逻辑、PathDecider决策逻辑、Speed Bounds Decider的ST边界构建进行源码级追踪分析，精确定位多障碍物场景下的性能瓶颈。

从SL层（横向路径边界）和ST层（纵向速度-时间）两个层面，系统识别Apollo现有算法在多障碍物场景下的核心缺陷：PathBoundsDecider逐障碍物贪心独立决策导致矛盾避让方向（BLOCKED）、安全裕度一刀切压缩通行空间、动态障碍物被完全排除在路径决策之外、路径-速度之间缺乏时空约束传递通道。通过具体数值示例进行定量分析。

**（3）改进算法设计与实现功能**

设计并实现基于时变最优通行带的改进算法，包含三项核心改进：

### 2.3 已完成的系统架构设计

改进后的系统采用SL/ST双层改进框架，沿Apollo现有的Scenario-Stage-Task三级架构进行最小侵入式集成：

**输入层**：接收Routing参考线、感知障碍物信息（含Prediction模块的预测轨迹）、车辆状态（位置、速度、加速度）。

**SL层改进（路径规划阶段）**：
- 到达时间估计：基于二阶运动学模型计算τ(s)
- 时变SL投影：利用预测轨迹计算动态障碍物在τ(s)时刻的SL位置
- 扫描线分组：按s_i^-升序排列，检测纵向重叠关系，划分连通分量
- 组内最优决策：按u_i排序，计算k+1个间隙，选最大间隙分配避让方向
- 路径边界构建：按分配的避让方向更新[l^-(s), l^+(s)]

**ST层改进（速度规划阶段）**：
- 时空可通行走廊定义：Ω = {(s,t) | g_{p*}(s) ≥ W_ego, t ≥ τ(s)}
- 走廊约束输出：对每个依赖动态障碍物移开的关键位置s_k，输出速度约束(s_k, τ(s_k))
- 与Speed Bounds Decider的ST边界在QP层面取最小值融合

**输出层**：生成ADCTrajectory轨迹（含位置、速度、加速度、时间戳），供下游控制模块使用。

### 2.4 已完成的核心算法设计

#### 2.4.1 时变最优通行带数学模型

**SL空间中的障碍物表示**：第i个障碍物在SL空间中投影为轴对齐矩形 O_i = [s_i^-, s_i^+] × [l_i^-, l_i^+]。引入综合安全裕度δ后，定义关键量 u_i = l_i^- - δ（右绕上界）和 v_i = l_i^+ + δ（左绕下界）。

**联合优化问题**：引入到达时间函数τ(s)后，问题建模为：
```
(d*, τ*) = arg max_{d,τ} min_s [l^+(s, d, τ) - l^-(s, d, τ)]
```
搜索空间为2^n（NP-hard），通过引理1和定理1将其高效降维。

**引理1（最优分割的连续性）**：按u_i值升序排列后，存在最优解d*使得分割是连续的，即存在分割点p使得排序中前p个障碍物左绕、后k-p个右绕。搜索空间从2^k降为k+1个候选分割点。证明方法：反证法，对任何不连续分割点构造新解d'，通过分析上界不变、下界不升来证明d'也是最优解。

**定理1（最大间隙最优性）**：对于分割点p，通行带宽度W(p) = g_p（第p个间隙值），因此最优通行带宽度 W* = max_{p=0,...,k} g_p。证明方法：对各分割点p直接代入验证，建立通行带宽度与间隙值的精确等式关系。

#### 2.4.2 基于扫描线分组的最优通行带求解算法（算法1）

完整的七步算法流程：
1. **到达时间估计**：基于二阶运动学模型 τ(s) = (-v_ego + √(v_ego² + 2a_ego(s - s_ego))) / a_ego
2. **时变SL投影**：查询预测轨迹，计算动态障碍物在τ(s)时刻的SL位置
3. **扫描线分组**：按s_i^-升序排列，维护s_max跟踪当前分量的最大s^+值，实现O(n log n)分组
4. **组内按u_i排序**：在每个连通分量内按u_i升序排列
5. **计算间隙并选最大**：计算k+1个间隙 g_p，选取arg max_p g_p
6. **构建路径边界**：按分配的避让方向更新路径上下界
7. **输出速度约束**：对依赖动态障碍物移开的位置s_k，输出时间有效性约束(s_k, τ(s_k))

**复杂度分析**：总时间复杂度O(n log n + K)，其中O(n log n)为障碍物处理，O(K)为路径边界构建（K为离散位置数）。

#### 2.4.3 多因子威胁度评估与动态安全距离

综合威胁度由五个因子加权求和：
```
Θ_i = w1·f_TTC(i) + w2·f_overlap(i) + w3·f_vel(i) + w4·f_type(i) + w5·f_inter(i)
```
默认权重：w1=0.30, w2=0.20, w3=0.15, w4=0.10, w5=0.25。

各因子定义：
- **碰撞时间因子f_TTC**：TTC_i = (s_i - s_ego)/(ṡ_ego - ṡ_i)，通过分段线性函数映射至[0,1]，T_crit=2s, T_max=7s
- **横向重叠率f_overlap**：衡量障碍物与自车行驶走廊在横向的重叠程度
- **相对速度因子f_vel**：使用Sigmoid函数 σ(v_rel,i / v_max) 平滑映射
- **障碍物类型因子f_type**：行人/自行车=1.0, 车辆=0.7, 静态障碍物=0.3
- **交互密度因子f_inter**：量化障碍物i与周围其他障碍物的聚集程度

动态安全距离：δ_i = δ_min + (δ_max - δ_min)·Θ_i，其中δ_max=1.4m（CRITICAL级），δ_min=1.05m（MINOR级）。

#### 2.4.4 时空可通行走廊与速度规划融合

**走廊定义**：Ω = {(s,t) | g_{p*}(s) ≥ W_ego, t ≥ τ(s)}

**走廊约束转化为QP线性约束**：对每个关键位置s_k，约束转化为 s(t) ≤ s_k, ∀t < τ(s_k)，即收紧ST上界：
```
s_j^ub ← min(s_j^ub, s_k), ∀j : t_j < τ_k
```

**双源融合**：走廊约束与Speed Bounds Decider的ST边界在QP层面取最小值融合：
```
s_j^ub = min(s_j^{ub,SBD}, s_j^{ub,corridor}), j = 0, 1, ..., K
```

该融合不改变QP问题的结构（决策变量数、Hessian矩阵H、等式约束矩阵A_eq均不变），仅修改上界向量x^ub的值，OSQP求解器的收敛性和计算效率不受影响。

## 三、存在的问题与解决方案

### 3.1 技术难点分析

**难点一：纵向部分重叠采用保守近似**

当障碍物在s方向仅部分重叠时（即存在某些s截面上只有部分障碍物活跃），当前算法将整个连通分量内所有障碍物均同时活跃为假设来计算间隙。这种保守近似在障碍物纵向间距较小时是合理的，但在某些特殊几何构型下可能低估实际可用空间。

**解决方案**：对s方向部分重叠的障碍物组，可沿s方向离散化为若干截面，在每个截面独立求解最优间隙后取全局最小值。这相当于将一维的组级决策扩展为逐截面决策，能够更精确地刻画通行带的纵向变化特征。该方案已作为未来优化方向列入研究计划。

**难点二：到达时间估计与速度规划的循环依赖**

到达时间函数τ(s)用于计算时变通行带，通行带约束影响速度规划，而速度规划结果反过来决定了自车的实际到达时间。这构成一个循环依赖。

**解决方案**：利用Apollo的周期性重规划机制（100ms一次）隐式耦合——每个周期使用上一周期的速度输出更新τ(s)，在连续多个规划周期中实现近似收敛。这种"单步延迟"策略在Apollo的路径-速度解耦框架中是标准做法，其在城市道路工况下的实际效果已通过实验验证。

**难点三：依赖预测轨迹的准确性**

动态障碍物的时变SL投影完全依赖Prediction模块输出的预测轨迹。在交互场景中，障碍物行为可能因自车决策而改变，当前的开环预测无法捕捉这种耦合效应。

**解决方案**：当前实现中取概率最高的轨迹进行投影。降级机制确保了当预测轨迹不可用或时间范围不足以覆盖τ(s)时，障碍物退化为静态处理，不劣于Baseline。更精细的处理方式（对多条轨迹分别计算时变投影并取并集）已规划为后续工作。

**难点四：走廊约束的QP接口设计**

走廊是连续的时空区域，需要将其离散化为一系列线性不等式约束，并确保与Apollo现有的OSQP求解器接口完全兼容。

**解决方案**：走廊约束直接转化为对ST上界的收紧：s_j^ub ← min(s_j^ub, s_k)，与Apollo原有的ST边界约束形式完全一致（均为s变量的上界约束），不改变QP问题结构，OSQP求解器的收敛性和计算效率不受影响。实验验证了该融合方案的有效性。

### 3.2 方法局限性

**（1）实验验证以仿真为主**

受限于实验条件，算法验证完全基于Apollo的PNC仿真环境，尚未在实车平台上进行测试。仿真环境无法完全模拟真实道路中的传感器噪声、感知延迟和执行误差，算法在实际部署中的鲁棒性有待进一步检验。

**（2）威胁度模型的参数敏感性**

五个威胁因子的权重（w1=0.30, w2=0.20, w3=0.15, w4=0.10, w5=0.25）基于经验设定并通过实验调优。尚未进行系统的参数敏感性分析，不同场景类型下的最优权重配置可能存在差异。

**（3）到达时间估计未与速度规划进行迭代修正**

到达时间函数τ(s)采用基于当前速度和加速度的二阶运动学模型，未与速度规划的输出进行迭代修正。当QP输出的速度曲线与τ(s)的估计偏差较大时（如需要大幅减速的场景），时变投影可能偏离实际情况。

### 3.3 改进与优化方案

针对上述局限性，提出以下改进方案：

| 问题 | 改进方案 | 优先级 | 计划时间 |
|------|--------|--------|--------|
| 参数敏感性 | 设计消融实验，逐一分析各权重因子对性能的影响 | 高 | 第10-11周 |
| 纵向部分重叠保守近似 | 实现逐截面决策的精确版本，对比保守近似的性能差异 | 中 | 论文讨论 |
| 到达时间循环依赖 | 利用上一周期速度输出进行τ(s)修正 | 中 | 已实现 |
| 仿真局限 | 在论文局限性部分充分讨论，提出实车验证的未来方向 | 低 | 论文撰写 |
